## Что разбирали (аудит)

Пользователь попросил фундаментально оценить, почему ассистент «плохо себя ведёт».
Прочитали код end-to-end (`agent.py`, `pipeline.py`, `tools.py`, policy `engine.py`,
eval `scoring.py`/`runner.py`/`simulator.py`/`llm_client.py`/`__main__.py`, `normalize.py`,
rule-файлы, `summary.py`/`state.py`/`termination.py`) + внешние best practices (2 субагента:
один — индустриальные источники с URL, второй — ширина кодовой базы + сверка TZ).

**Вывод аудита:** «плохое поведение» — не набор багов, а следствие **двух корневых причин**,
а метод фиксов (добавлять правило в монолитный промпт на каждый симптом) сам по себе —
антипаттерн (context rot):
1. **Модель ниже порога надёжности** (haiku) для многоходового voice-агента с tool-calling.
   D-11 («haiku держит роль 3/3») дал ложную уверенность — тот A/B был одноходовым без инструментов.
2. **Монолитный single-prompt агент** (`build_system_prompt` + все инструменты каждый ход +
   7-правильный `_POLICY_PREAMBLE`).
- Мета-проблема: text-edge eval **слеп** к аудио/STT/termination-backstop — ровно к тому, что
  сломало реальный звонок.

## Решения, добавленные в эту сессию

- **D-15** в `DECISIONS.md` — направление: прекратить symptom-patching; перейти на flow-граф
  (Pipecat Flows), поднять модельный пол (Gemini 2.5 Flash / GPT-4.1, config-only по D-5),
  termination структурно (`end_conversation`), grounding над-обещаний, Nova-3 keyterms (STT),
  аудио-уровень eval, фокус на compliance/ров (D-7). Секвенс P0→P1→P2→P3; исполнение через
  brainstorm→спека→план. **D-11 переоткрыт.**

## Прерогатива среды (ВАЖНО для исполнителя)

Часть задач **нельзя выполнить без окружения владельца**:
- **P0.3** (модельный A/B) — нужны реальные ключи Gemini/OpenAI/Anthropic в `.env`.
- **S1** (спайк) — `uv sync --extra call` + сеть (загрузка `pipecat-flows`, smart-turn ONNX).
- Прогоны eval тратят деньги (sim+judge ~$0.05–0.13 за свип).

Поэтому исполнитель-агент делает **код-задачи без провайдеров** (P0.1, P0.2, T1; T2/T3 после
того как владелец прогонит S1), а **P0.3 и S1 запускает владелец** по готовым командам из плана.
